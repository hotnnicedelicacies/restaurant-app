## Quick orientation

- **What this is.** A from-scratch rebuild of <https://hotnnicedelicacies.com> — a Middlesbrough home-kitchen meal delivery service. Italian classics + West African home cooking. UK only.
- **Stack.** Next.js 16 (app router) · React 19 · Tailwind v4 · Supabase (DB + Auth + Storage) · Stripe (checkout + webhooks) · Resend (email) · Vercel (hosting). Sonner toasts. Zustand for cart. Radix/shadcn-ui primitives kept from the original scaffold and recustomised.
- **Design system.** "Walnut & Cream" — Cormorant Garamond serif + Geist Mono labels. Cream `#F1E5CD`, walnut `#2D1F18`, bronze `#A56F40`. Heritage British editorial feel. Inspired by Hawksmoor / Daylesford / St. JOHN.
- **The source of truth** for design + data model is in `design-explorations/` (HTML mockups + `brand-spec.md` + `admin-settings.md`). 33 HTML mockup pages live there, fully approved and reviewed. The Next.js build ports them faithfully.

## Key design decisions to honour

1. **PDF invoices: NOT implemented in v1.** User signed off on HTML email + `/receipt/[ref]` page with browser-printable styles. If a customer needs a PDF, they print from the browser. Don't add Puppeteer.
2. **Bronze (`#A56F40`) is decorative-only on cream**. Fails AA for body text. Use `bronze-deep` (`#7E5530`) when you need bronze-toned text. See `design-explorations/brand-spec.md` for the contrast contract.
3. **Anti-slop discipline.** No purple gradients. No emoji icons. No Inter for display. No SVG illustrations of food. No filler verbs ("Elevate", "Seamless"). Heritage prose voice.
4. **Image storage: Supabase Storage** (`menu-images` bucket, public, 10 MB limit). Use `getStorageUrl(path)` from `lib/supabase/storage.ts` everywhere — one swap-point for R2 migration later if needed.
5. **Order ref format**: `HNN-XXXX-NNNN` (4 chars + 4 digits, ambiguous chars omitted). Generated by the Postgres function `generate_order_ref()`. The ref IS the access token for `/track/[ref]`, `/receipt/[ref]`, `/confirmation/[ref]` — no login required to view by ref.

## Critical gotchas

1. **Tailwind v4 + @theme tokens.** Use named utility classes (`bg-walnut`, `text-cream`) — NOT `bg-[--color-walnut]` which the user hit and reported as broken. The sed fix at this commit converted them all.
2. **The legacy menu page at `app/menu/page.tsx` was REPLACED.** `constants/meals.ts` is still imported by `lib/data/menu.ts` as a fallback when Supabase isn't reachable — don't delete it.
3. **AppWrapper is GONE.** I removed it from root layout. Old pages that used it (e.g. existing `app/order/page.tsx` — which is the OLD form-based order flow) will render without header/footer. That's intentional; `/order` is deprecated by `/checkout`. Delete the route when convenient.
4. **`components/Header.tsx`, `components/Footer.tsx`, `components/MealCard.tsx`, `components/FoodHygieneBadge.tsx`, etc.** are still on disk but unused. Safe to delete after Phase 6 visual review.
5. **`app/order/` route is unused** but the email-template helpers in `lib/orders/emailTemplates.ts` + `lib/orders/sendOrderEmails.ts` + `lib/orders/timeSlots.ts` may be useful for Phase 5 email work — read before reimplementing.
6. **Two Tailwind utility prefixes that look strange but are correct**:
   - `border-rule` / `bg-rule` — uses the `--color-rule` token (rgba walnut at 0.18 alpha)
   - `text-ink-muted` — uses `--color-ink-muted` (deep walnut-brown for body italic on cream)
7. **Stripe webhook events** to register in the dashboard: `payment_intent.succeeded`, `payment_intent.payment_failed`, `charge.refunded`, `charge.dispute.created`. URL: `/api/stripe/webhook`. Local dev: `stripe listen --forward-to localhost:3000/api/stripe/webhook`.

## Launch checklist (Phase 7 cutover)

Pre-deploy:

1. **Environment variables** on Vercel (production + preview):
   - `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`, `SUPABASE_SECRET_KEY`
   - `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY`, `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`
   - `RESEND_API_KEY`, `ORDER_FROM_EMAIL`, `ORDER_NOTIFICATION_EMAIL`
   - `NEXT_PUBLIC_SITE_URL` (set to live domain so sitemap + emails use absolute URLs)
2. **Supabase**: ensure all 3 migrations are applied (`supabase/setup.sql` is the combined paste-once version). Confirm RLS is on. Verify `menu-images` storage bucket exists + is public.
3. **Stripe webhook** at `https://hotnnicedelicacies.com/api/stripe/webhook` registered for: `payment_intent.succeeded`, `payment_intent.payment_failed`, `charge.refunded`, `charge.dispute.created`. Capture the signing secret into `STRIPE_WEBHOOK_SECRET`.
4. **Resend domain** verified for `orders@hotnnicedelicacies.com` (DNS records: SPF, DKIM). Without this, emails go to spam or fail outright.
5. **Admin user** seeded — `hotnnicedelicacies@gmail.com` is_admin=true (the seed `--create-admin` flag handles this).
6. **OG image** — replace the placeholder reference at `/og-image.jpg` with a real 1200×630 image. Apple icon at `/apple-icon.png`.
